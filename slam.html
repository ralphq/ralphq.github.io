SLAM (Georgia Tech)

Drone Navigation and Mapping in AI for Robotics course

Background

SLAM (Simultaneous Localization and Mapping) is a fundamental problem in robotics where an agent must simultaneously estimate its own position while mapping an unknown environment using sensor observations. This creates an interdependent estimation challenge where accurate localization requires a reliable map, while accurate mapping requires precise knowledge of the agent's position. In this project, I implemented a GraphSLAM system for autonomous drone navigation in an environment containing trees of unknown locations and sizes. In the above visualizations, the actual trajectory is plotted in red and the estimated trajectory in blue.

The system addressed two primary objectives: first, to estimate the drone's trajectory and construct a map of tree locations using noisy sensor measurements, and second, to navigate to a specified target location while avoiding collisions with mapped obstacles. The drone's sensors provided distance, bearing, and size measurements of trees within a limited detection radius, with both measurement and movement commands subject to Gaussian noise. Trees would appear and disappear from the sensor's field of view as the drone moved through the environment, and the complete system required execution within a 10-second computational time constraint.

Takeaways

Coordinate System Management: Converting between polar sensor measurements and Cartesian world coordinates while maintaining consistency across timesteps proved challenging. Accumulated orientation error from noisy steering commands significantly impacted tree position estimation accuracy over time.

This required balancing measurement uncertainty with geometric constraints to prevent duplicate map entries.

Engineering Trade-offs: Given the 10-second time constraint, simple heuristics often outperformed sophisticated algorithms, emphasizing the importance of computational efficiency in real-time robotics applications.
